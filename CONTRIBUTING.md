# Contributing to showittome

Thank you for your interest in contributing to this educational security demonstration project.

## Important Notice

This repository contains intentionally malicious code for **educational purposes only**. All contributions must align with the project's educational mission and the [Hippocratic License](LICENSE.md).

## What We Accept

**Acceptable contributions:**
- Documentation improvements
- Additional defensive guidance and security recommendations
- Bug fixes in the demonstration infrastructure (Docker, Flask API)
- Translations of educational content
- Additional examples of attack patterns (with clear educational framing)

**We do NOT accept:**
- Improvements that make the attack code more effective or stealthy
- Code that targets additional credential types or systems
- Obfuscation or evasion techniques
- Any code intended for malicious use

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b feature/your-feature`)
3. **Make your changes**
4. **Test locally** - Ensure Docker builds and the API runs
5. **Submit a Pull Request** with a clear description of changes

## Code Style

- **Python**: Follow PEP 8
- **R**: Follow tidyverse style guide
- **Documentation**: Use clear, accessible language

## Reporting Security Concerns

If you have concerns about potential misuse of this code or discover a security issue, please:

1. Open a GitHub issue, or
2. Contact the maintainer directly

## Questions?

Open an issue for any questions about contributing.
