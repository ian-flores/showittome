# Security Policy

## Important Notice

This repository contains **intentionally malicious code for educational purposes only**. The code demonstrates supply chain attack techniques to help security professionals and data scientists understand these threats.

## Reporting Security Issues

### For This Repository

If you discover a security issue in the demonstration infrastructure (Docker configuration, Flask API, etc.), please:

1. Open a GitHub issue with the `[SECURITY]` prefix
2. Describe the issue and potential impact
3. Suggest a fix if possible

### For Misuse Concerns

If you observe this code being used maliciously or have concerns about potential misuse:

1. Open a GitHub issue describing your concern
2. Contact the maintainer directly
3. If appropriate, report to relevant authorities

## Scope

This security policy covers:
- The demonstration infrastructure (Docker, Flask API, Nginx)
- Documentation and educational content
- Repository configuration

This policy does **not** apply to:
- The intentionally malicious R package code (that's the point of the demo)
- Third-party dependencies (report those upstream)

## Responsible Use

By using this repository, you agree to:
- Only use the code for educational purposes or authorized security testing
- Never deploy against systems without explicit permission
- Comply with the [Hippocratic License](LICENSE.md)
- Follow all applicable laws and regulations
